Slideshot

Hotkey-driven screenshot annotation and session recording, built for feeding process documentation into a Claude Project.

What it does

Slideshot lets you mark up a live web page — highlight fields, flag dropdowns, add callouts and notes — then export an annotated screenshot (plus a matching full-page version and a notes file) with one hotkey. For multi-step processes, a session groups several captures together, optionally with voice narration, and exports everything as one PDF + zip bundle sized to fit a single Claude Project upload.

Hotkey reference

Capture basics

KeyDoes
Shift+1Enter capture mode. Press again while already in capture mode to open the area selector; drag to draw the capture region, then Enter/Y to confirm.
EscapeCancel capture mode entirely with no output — for when it's entered by mistake. (Also cancels an in-progress area selection or dropdown crop, one step at a time.)
Ctrl+ZUndo the last marking action (highlight, context tag, number, description/overview).

Highlight types (while in capture mode, hover an element then press)

KeyDoes
Shift+GSmall highlight — a solid box around the hovered element.
Ctrl+Shift+GKey field highlight — a jagged/zigzag outline, for flagging something especially important.
Shift+HCallout — a text box with an arrow pointing at the hovered element. Requires you to type text before it's created; cancelling discards it.
Shift+DDropdown flag — see the dedicated section below. Works even before Shift+1.
Shift+TText annotation with an arrow, attachable to any existing highlight.

Repeat the same hotkey on the same element to toggle a highlight off instead of creating a duplicate.

Editing a highlight

KeyDoes
Shift+JDescribe the highlight under the cursor — set/edit its title and free-text description.
Shift+NToggle number mode, then press 0–9 over a highlight to tag it with that number.
Shift+CTag the hovered element as "additional context" — an auto-captured label + text snippet, no typing needed.
Shift+KAdd a top-level overview note for the whole capture, not tied to one highlight.

Drag any number/context badge, or any callout/text-annotation box, to reposition it — the arrow snaps to one of 10 preset anchor points (corners, side-centers, offset-left/right) on the target element.

Sessions & narration

KeyDoes
Ctrl+Shift+EStart or stop a session. Every capture finished while a session is recording is added to it as a step (its individual files aren't downloaded separately — see Sessions below).
Alt (hold)Push-to-talk narration while a session is recording. Speech is transcribed and time-aligned to whichever step it was held during.

The dropdown-flag tool, explained

Open dropdowns and hover menus are fragile: the moment your mouse moves toward Slideshot's own UI, the browser drops the element's hover state and the menu closes — before a normal highlight could ever capture it. Shift+D is a separate capture method built specifically for this:

  1. Hover the open dropdown and press Shift+D. This works whether or not you've entered capture mode yet.
  2. The whole viewport is screenshotted immediately — before any visual change happens on screen — so the open dropdown is frozen into the image no matter what happens next.
  3. That screenshot is shown full-screen so you can drag a box around just the dropdown; releasing crops it out as its own small image file.
  4. A dashed marker is drawn on the main capture over the exact area the dropdown occupied, so the two images are obviously linked when viewed side by side. The small crop itself is left unmodified (no border baked on), in case you want to stitch it into something else manually later.

Dropdown-flag marker (bolder dash) vs. callout marker (finer dash) — visually distinct so you can tell them apart on a capture.

Limitation: this only works for dropdowns built from normal page elements (custom/ARIA menus, CSS :hover panels). A native HTML <select> renders its open list as an OS-level control outside the page's pixels — no browser extension can screenshot that, open or closed.

Sessions

A session (Ctrl+Shift+E) groups multiple captures — across pages, if needed — into one export instead of handling each capture as a one-off. While a session is recording:

  • Each finished capture becomes a step. Its files are not downloaded individually — they'd just be redundant clutter once the session's own export exists.
  • A collapsible on-page panel (bottom-right of the page) shows session status, the step list, and lets you Start/Stop/Discard the session, plus Delete or edit the narration text for any step.
  • The toolbar popup (click the Slideshot icon) shows the same step list and is where you Export.

Hold Alt to narrate while recording — release it (or switch away from the window) to stop. Narration is transcribed locally in the browser via speech recognition (no audio is ever saved, only the transcribed text) and time-aligned to whichever step was current when you were talking.

Export

Click Export in the toolbar popup after stopping the session. This produces one zip download containing:

  • A PDF (split into session-part1.pdf, session-part2.pdf, … if the session is large enough to near a single-file upload limit) — one page per step, with that step's screenshot(s), highlight notes, and narration. This is the file meant for a single upload into a Claude Project.
  • session-notes.md — the same text content in plain markdown.
  • Raw PNGs for every step (cropped + full-page + any dropdown crops) — a lossless backup, and useful if you want to reuse an image directly rather than re-extract it from the PDF.

Example session flow

  1. Navigate to the first screen of the process you're documenting. Press Ctrl+Shift+E to start a session — an on-page toast confirms it, and the session panel appears bottom-right.
  2. Press Shift+1 to enter capture mode. Hover a field and press Shift+G to mark it; press Shift+J to give it a title/description if the auto-detected one isn't right.
  3. Notice a dropdown you need to show expanded. Hover it and press Shift+D — drag a box around it in the frozen screenshot that appears, then release.
  4. Hold Alt and narrate what you're doing on this screen; release when done.
  5. Press Shift+1 again, drag the capture area, press Enter — this step's screenshot, highlights, dropdown crop, and narration are all added to the session as one step.
  6. Navigate to the next screen in the process and repeat steps 2–5 for as many steps as the process needs.
  7. When finished, press Ctrl+Shift+E again to stop the session.
  8. Click the Slideshot toolbar icon, confirm the step list looks right, and click Export.
  9. Upload the resulting PDF (all parts, if split into more than one) to your Claude Project to generate the PowerPoint.
